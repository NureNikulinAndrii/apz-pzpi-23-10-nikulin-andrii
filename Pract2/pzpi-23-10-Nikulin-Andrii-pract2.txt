3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Загальні відомості про систему
• Назва системи: Discord (платформа реального часу для спілкування).
• Розробник: Discord Inc.
Призначення системи: Надання послуг голосового, відео- та текстового зв'язку для великих спільнот (геймерів, розробників, освітніх груп). Основні функції включають створення серверів (guilds), обмін повідомленнями в реальному часі, голосові канали з низькою затримкою та підтримку ботів для автоматизації.
3.2 Архітектурний стиль системи
      Система використовує мікросервісну архітектуру. Вибір обґрунтований необхідністю незалежного масштабування компонентів: шлюз (Gateway) обробляє мільйони одночасних WebSocket-з'єднань, тоді як голосові сервери займаються лише трафіком WebRTC. Це забезпечує високу відмовостійкість: збій одного сервісу не призводить до зупинки всієї платформи.
3.3 Основні компоненти системи
• Інтерфейс користувача: Кросплатформні додатки (Desktop, Mobile, Web), що взаємодіють з бекендом через API та WebSocket.
• Серверна частина (Gateway): Шлюз на базі Elixir/Erlang для підтримки постійних з'єднань та аутентифікації користувачів.
• Рівень бізнес-логіки: Набір мікросервісів на Rust, Go та Python, що відповідають за керування правами доступу, обробку команд та модерацію.
• Рівень зберігання даних: Високопродуктивна NoSQL база даних ScyllaDB (раніше Cassandra) для зберігання мільярдів повідомлень та Redis для кешування сесій.
• Допоміжні сервіси: Voice Servers (C++/Rust) для обробки аудіо та Event Bus (Kafka) для маршрутизації подій


3.4 Взаємодія компонентів
1. Взаємодія побудована на подієво-орієнтованій моделі (Pub/Sub).
2. Клієнт встановлює WebSocket-з'єднання з Gateway.
3. При відправці повідомлення подія передається через брокер Kafka.
4. Gateway "пушить" оновлення всім підписаним клієнтам у реальному часі, що мінімізує затримку порівняно з HTTP-опитуванням.
3.5 Масштабованість та надійність
• Балансування навантаження: Використовується стратегічна маршрутизація трафіку для розподілу мільйонів користувачів між Gateway-вузлами.
• Відмовостійкість: Реалізовано механізми session resume та exponential backoff для швидкого відновлення з'єднання при мережевих збоях.
• Кешування: Redis забезпечує миттєвий доступ до станів користувачів, розвантажуючи основні бази даних.
• Реплікація: Використовується ScyllaDB з налаштованим фактором реплікації (RF) для забезпечення цілісності історії чатів.
• Шардинг: Дані та сесії розподіляються за шардами (наприклад, по серверах/guilds), щоб уникнути перевантаження окремих вузлів.
3.6 Приклади програмної реалізації
     Для демонстрації архітектурних принципів взаємодії з Gateway було розроблено бота на мові C#. Нижче наведено промт, за допомогою якого було згенеровано програмний код для цього завдання.
     Промт для створення коду бота:
     "Напиши код для Discord-бота на C# з використанням бібліотеки Discord.Net. Бот повинен реалізувати архітектурний патерн асинхронної обробки подій через Gateway. Необхідно налаштувати GatewayIntents для читання повідомлень, реалізувати логування та створити обробник події MessageReceived, який відповідає текстом 'Pong! Gateway працює.' на команду '!ping'. Код має демонструвати використання асинхронності (Task) та принцип обробки подій у реальному часі."

Промпт (запит) до ШІ Gemini: " "Напиши код для Discord-бота на C# з використанням бібліотеки Discord.Net. Бот повинен реалізувати архітектурний патерн асинхронної обробки подій через Gateway. Необхідно налаштувати GatewayIntents для читання повідомлень, реалізувати логування та створити обробник події MessageReceived, який відповідає текстом 'Pong! Gateway працює.' на команду '!ping'. Код має демонструвати використання асинхронності (Task) та принцип обробки подій у реальному часі."
1   using System;
2   using System.Threading.Tasks;
3   using Discord;
4   using Discord.WebSocket;
5   
6   namespace MyDiscordBot
7   {
8       class Program
9       {
10          static async Task Main(string[] args)
11          {
12              var bot = new DiscordBotClient();
13              await bot.StartAsync();
14          }
15      }
16  
17      class DiscordBotClient
18      {
19          private DiscordSocketClient _client;
20  
21          public async Task StartAsync()
22          {
23              // Налаштування клієнта (вмикаємо можливість читати повідомлення)
24              var config = new DiscordSocketConfig 
25              { 
26                  GatewayIntents = GatewayIntents.AllUnprivileged | GatewayIntents.MessageContent 
27              };
28              
29              _client = new DiscordSocketClient(config);
30  
31              _client.Log += LogAsync;
32              _client.Ready += ReadyAsync;
33              _client.MessageReceived += MessageReceivedAsync;
34  
35              // токен
36              string token = "MTQ5NTA2Njk0MTY5Njc3MDA2OA.GAAu9N.V0ZXj7k0v35e-8-Irm0YyLyHUmNJZXbInZSVP4";
37  
38              await _client.LoginAsync(TokenType.Bot, token);
39              await _client.StartAsync();
40  
41              await Task.Delay(-1); // Тримаємо програму відкритою
42          }
43  
44          private Task LogAsync(LogMessage log)
45          {
46              Console.WriteLine(log.ToString());
47              return Task.CompletedTask;
48          }
49  
50          private Task ReadyAsync()
51          {
52              Console.WriteLine($"Підключено до Gateway як {_client.CurrentUser}");
53              return Task.CompletedTask;
54          }
55  
56          private Task MessageReceivedAsync(SocketMessage message)
57          {
58              // Бот ігнорує власні повідомлення
59              if (message.Author.Id == _client.CurrentUser.Id)
60                  return Task.CompletedTask;
61  
62              // Якщо хтось пише !ping, бот відповідає
63              if (message.Content == "!ping")
64              {
65                  Console.WriteLine($"Отримано команду від {message.Author.Username}");
66                  message.Channel.SendMessageAsync("Pong! Gateway працює.");
67              }
68  
69              return Task.CompletedTask;
70          }
71      }
72  }
